> A comprehensive elderly-care companion app (نظام رعاية المسنين) built with Flutter.
> Wanas connects elderly residents, their families, and care professionals through
> medication reminders, AI companionship, real-time communication, and care management.

## Overview

Wanas is a multi-role, Arabic-first (RTL) mobile application that bridges elderly
individuals, their families, and the care team around them. The app ships a tailored
experience for **six roles**:

| Role | Arabic | Focus |
|------|--------|-------|
| Elderly | المسن | Home, medications, AI companion, calls, memories, activities |
| Family | الأسرة | Resident linking, dashboards, shared media, communication |
| Nurse | الممرض | Operations, reports, medication administration |
| Specialist | الأخصائي | Assessments, activities, complaints, KPIs |
| Volunteer | المتطوع | Profile, scheduling, assignments |
| Admin | المدير | Facility settings, residents, staff management |

### Key features

- **AI Companion** — voice-driven conversation (TTS + speech-to-text) for daily check-ins
- **Medication Management** — reminders and adherence tracking via local notifications
- **Family Bridge** — shared media, memories, and updates between family and residents
- **Real-time Communication** — chat and notifications over WebSocket (Socket.IO)
- **Reports & PDF** — generate and print care reports
- **Secure auth** — AWS Cognito with biometric unlock and secure token storage

## Tech Stack

- **Framework:** Flutter 3.x · **Language:** Dart 3.x
- **State Management:** [Riverpod](https://riverpod.dev) (`flutter_riverpod`)
- **Backend API:** [raaya-backend](https://github.com/amrhanygomaa/raaya-backend) (NestJS, REST + WebSocket)
- **Authentication:** AWS Cognito + `flutter_secure_storage` + `local_auth` (biometrics)
- **Push Notifications:** Firebase Cloud Messaging
- **Voice/AI:** `flutter_tts`, `speech_to_text`, `audioplayers`, `record`
- **Localization:** `flutter_localizations` (Arabic, RTL)

## Quick Start

### Prerequisites
- Flutter SDK ≥ 3.0.0 and Dart SDK ≥ 3.0.0
- Android Studio / Xcode for emulators
- A running [raaya-backend](https://github.com/amrhanygomaa/raaya-backend) instance (or use the hosted API)

### Installation

```bash
git clone https://github.com/amrhanygomaa/wanas.git
cd wanas
flutter pub get
flutter run
```

### Configuration

This app reads configuration at **compile time** via `--dart-define` (see
[`lib/config/api_config.dart`](lib/config/api_config.dart)). No `.env` file is used at runtime.

| Define | Description | Default |
|--------|-------------|---------|
| `API_BASE_URL` | Backend base URL | `https://api.helpers-tech.com` |
| `ADMIN_REG_SECRET` | Secret required to register a new facility's first admin | _(empty)_ |
| `FACILITY_ID` | Default facility for single-tenant builds | _(empty)_ |

Example:

```bash
flutter run \
  --dart-define=API_BASE_URL=http://10.0.2.2:3000 \
  --dart-define=ADMIN_REG_SECRET=your-secret
```

> **Firebase:** `lib/firebase_options.dart` contains the standard client-side
> Firebase configuration generated by `flutterfire configure`. Push notifications
> require a valid `google-services.json` (Android) / `GoogleService-Info.plist` (iOS),
> which are **not** committed. See [Firebase setup](https://firebase.flutter.dev/docs/overview).

### Build for production

```bash
flutter build apk --release          # Android APK
flutter build appbundle --release    # Android App Bundle
flutter build ios --release          # iOS (macOS only)
```
